## ðŸ§ª Lab Philosophy

### Beginner Labs: Observation & Interaction
**Goal:** Build intuition without code

Examples:
- "Ask ChatGPT about yesterday's news" (knowledge cutoff)
- "Make AI hallucinate" (Reginald P. Crustworth)
- "Compare GPT-4 vs Claude responses"

**Characteristics:**
- No coding required
- Immediate feedback
- Pattern recognition
- "Aha!" moments

---

### Intermediate Labs: Guided Building
**Goal:** Learn by doing with support

Examples:
- "Build a chatbot with OpenAI API" (step-by-step)
- "Add image generation to your app" (copy-paste code)
- "Deploy to Vercel" (guided tutorial)

**Characteristics:**
- Step-by-step instructions
- Copy-paste friendly code
- "Change X and observe Y" experiments
- Working demo as output

---

### Advanced Labs: Independent Projects
**Goal:** Real developer experience

Examples:
- "Build a RAG system for your company docs" (requirements only)
- "Implement prompt injection defense" (research required)
- "Fine-tune a model on your dataset" (debugging expected)

**Characteristics:**
- Requirements-only (no step-by-step)
- Multiple solution paths
- Debugging is part of the learning
- Production-quality expected

---

### Expert Labs: Open-Ended Challenges
**Goal:** Ship real products

Examples:
- "Build this platform with these constraints"
- "Contribute a new feature to our open source repo"
- "Optimize inference latency under 100ms"

**Characteristics:**
- Open-ended problem
- No provided code
- Research and experimentation required
- Code review by maintainers
- Production deployment required
