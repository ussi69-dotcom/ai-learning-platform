## üîå What is Workflows Agent?

**Workflows Agent** is Microsoft 365 Copilot's conversational interface for building automations. It works alongside App Builder (for creating lightweight apps) and sits on the Power Platform infrastructure (but uses a different build experience than the classic Power Automate designer). Think of it as:

> **"ChatGPT that can actually DO things in your Microsoft 365 environment."**

### The Three Pillars

<ConceptCard title="The Agent Trinity" icon="‚ö°">

Every Workflows Agent operates on three fundamental components:

1. **Trigger (The Ear)** - What event wakes the agent up?
   - Email arrives from specific sender
   - File added to SharePoint folder
   - Teams message contains keyword
   - Scheduled time (daily at 9 AM)

2. **Action (The Hands)** - What does the agent DO?
   - Search documents across OneDrive
   - Draft email responses
   - Update Excel spreadsheets
   - Create Planner tasks
   - Post Teams messages

3. **Response (The Voice)** - How does it report back?
   - Confirmation message
   - Summary of actions taken
   - Error handling explanation
   - Request for human approval

</ConceptCard>

## üõ°Ô∏è Loop Prevention & Safety

**The Infinite Email Problem:**

What happens if your agent sends a Teams message that triggers... another workflow... that sends an email... that triggers the first workflow?

**üí• Infinite loop. 1000 notifications in 10 seconds.**

### The Two Safety Mechanisms

<Steps>

#### 1. Outlook Rules for Agent-Sent Messages

Create an Outlook rule:
- **IF** sender is "Workflows Agent" (or your automation account)
- **THEN** skip all other rules and move to "Bot Messages" folder

#### 2. Workflow-Level Circuit Breakers

In your workflow prompt, add:

```text
IMPORTANT: Before sending any message, check if the triggering event was caused
by a previous automation (look for tag "[AUTO]" in subject).
If yes, STOP and log "Loop prevented" instead of executing.
```

</Steps>

<Callout type="warning">
**Production Rule:** NEVER have two workflows that can trigger each other without explicit loop prevention logic.
</Callout>

The safest automations insert a **human checkpoint** before irreversible actions.

## üìä Managing Your Workflows

### The Activity Tab

Every workflow has an **Activity** log showing:

<MDXImage src="images/workflow-activity-screen.png" alt="Workflow Activity monitoring" caption="The Activity tab shows every workflow run with status, trigger, and actions taken" />

| Column | Information |
|--------|-------------|

<Callout type="tip">
**Pro Debugging Tip:** Add a "Notify me on failure" step that sends a Teams message with the error log when something breaks. Don't wait for users to report issues.

### Testing Workflows

Before activating a workflow in production:

<Steps>

#### 1. Test Mode
Run the workflow with sample data (Copilot provides a "Test run" / beaker view)

**Tip:** In Workflows Agent, the test beaker view shows step-by-step execution; re-run tests after each prompt change.

#### 2. Dry Run
Add a step: "Instead of sending email, save draft to 'Test Drafts' folder"

#### 3. Limited Scope
Start with trigger filter: "Only emails from myself for first 24 hours"

#### 4. Monitor First Week

</Steps>

<Callout type="warning">

### The 3 Launch Rules

<Steps>

#### 1. Narrow Scope First

#### 2. Draft-Only External Messages

## üîí Security & Governance

**"Great, I just built a bot that can read my CEO's email and post to social media."**

**Stop.** Let's talk about safety rails.

### The Three Guardrails

<ConceptCard title="Agent Security Model" icon="üõ°Ô∏è">

#### 1. Permission Inheritance
Workflows Agent operates under **YOUR** permissions, not elevated admin rights.

- ‚úÖ If you can't access the HR folder, neither can your agent
- ‚úÖ If you can't send as CEO, neither can your agent
- ‚ùå Agents CANNOT escalate privileges

#### 2. Admin Controls
IT administrators can:
- Disable Workflows Agent for specific users/groups
- Require approval for workflows that access sensitive data
- Audit all agent activities via Microsoft 365 Compliance Center
- Set data loss prevention (DLP) policies blocking certain actions

#### 3. Transparency & Audit
- Configure email footers to indicate automated messages (recommended: "Sent via Copilot Automation")
- Add "[AUTO]" tags to subjects for easy filtering
- Activity logs are retained per your tenant's audit log policies

</ConceptCard>

Think of governance as **layered defense**, not a single toggle.

<Diagram type="ai-security-layers" />

### The 3 Rules of Agent Governance

1. **üõ°Ô∏è Least Privilege** ‚Äî Does your receipt-filing agent REALLY need Write access to the entire OneDrive?  
   **Fix:** Limit to specific "Receipts" folder only.

2. **‚úÖ Human-in-the-Loop for High Stakes** ‚Äî Actions with consequences (sending money, deleting files, external communications) should require manual approval.  
   **Example:** Email drafts should save as drafts, not auto-send.

3. **üîí Transparent Identification** ‚Äî The agent must identify itself as AI, not impersonate a human.  
   **Bad:** Email signature "Sarah from Support Team"  
   **Good:** Email signature "Workflows Agent on behalf of Support Team"
